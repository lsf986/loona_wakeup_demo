## 1. 项目概述

本项目是 **Loona 机器人语音唤醒模块** 的 Python 实现，用于桌面办公场景下的语音唤醒检测。该模块支持三种唤醒触发模式：

- **KWS模式** (Keyword Spotting): 使用预训练唤醒词模型（如 "hey_jarvis"）
- **VAD模式** (Voice Activity Detection): 无唤醒词，通过语音活动检测触发
- **多模态模式** (Multimodal): 结合音频、视觉（人脸/唇动/注视）、空间定位（DOA）等多维度证据

## 3. 核心模块详解

### 3.1 音频前端 (perception_audio.py)

#### 3.1.1 高通滤波器 (HighPass1)

```python
class HighPass1:
    """1阶高通滤波器，截止频率80Hz"""
    # 作用：过滤风扇/空调等低频稳态噪声
    # 公式: y[n] = a * (y[n-1] + x[n] - x[n-1])
```

**工作原理：**
- 截止频率: 80Hz
- 用途: 去除风扇、空调等低频轰鸣噪声
- 避免稳态噪声被误判为人声

#### 3.1.2 浊音检测 (autocorr_pitch)

```python
def autocorr_pitch(x, sr=16000, fmin=80.0, fmax=400.0) -> (pitch_hz, voiced_strength)
```

**工作原理：**
1. 计算归一化自相关函数
2. 在80-400Hz范围内寻找峰值（人声基频范围）
3. 返回基频和浊音强度(0-1)
4. 判断条件: `voicing ≥ 0.22` 且 `pitch > 0` 且 `rms > 80`

**作用：** 区分人声与键盘敲击等瞬态噪声

#### 3.1.3 频谱重心 (spectral_centroid)

```python
def spectral_centroid(x, sr=16000) -> float
# 计算: Σ(freq * magnitude) / Σ(magnitude)
```

**原理：**
- 近场人声: 300-3500Hz (高频成分丰富)
- 远场语音: 重心偏低 (高频被空气吸收)
- 用于区分用户近讲与同事远场交谈

#### 3.1.4 GCC-PHAT 声源定位 (DOA)

```python
def gcc_phat(sig, ref, sr=16000, max_tau=None) -> (tau_sec, strength)
```

**工作原理：**
1. 计算两通道信号的互相关
2. 相位变换(PHAT)加权，增强尖锐峰值
3. 通过插值提高时间分辨率
4. 计算到达时间差: `τ = argmax(R)`
5. 转换为角度: `θ = arcsin(c * τ / d)`
   - c: 声速 (343 m/s)
   - d: 麦克风间距 (默认 4cm)

**用途：** 判断说话者是否在正前方（±30°锥形区域）

#### 3.1.5 近场检测 (near_field_ok)

```python
near_field_ok = (
    rms >= near_field_rms          # 能量门限（默认400-900）
    and centroid ∈ [300, 3500]     # 频谱范围
)
```

**双重判断：**
- 绝对能量: 近场语音RMS显著高于远场
- 频谱特征: 远场高频衰减

### 3.2 视觉前端 (perception_visual.py)

#### 3.2.1 人脸检测

- **算法:** OpenCV Haar Cascade
- **模型:** `haarcascade_frontalface_default.xml`
- **优化:** 每2帧检测一次，中间帧复用结果（降低50% CPU）
- **参数:** `scaleFactor=1.15`, `minNeighbors=3`

#### 3.2.2 四项视觉门限

| 门限 | 计算方法 | 阈值 | 用途 |
|------|----------|------|------|
| face_present | 人脸检测结果 | 检测到人脸 | 确认有人 |
| gaze_aligned | 人脸中心水平偏移 | \|offset\| ≤ 0.32 | 用户注视设备 |
| near_field | 人脸框面积/画面面积 | ≥ 0.02 (2%) | 距离足够近 |
| lip_moving | 口部ROI帧间差标准差 | std ≥ 2.0 | 正在说话 |

**唇动检测细节：**
1. 提取口部ROI: 脸部下方58%-90%，左右22%-78%
2. 缩放到60×30像素，计算帧间绝对差
3. 维护最近25帧（约1秒）的历史
4. 判断: `std ≥ 2.0` 且 `recent_mean ≥ 1.2`

### 3.3 环形缓冲 (RingBuffer)

```python
class RingBuffer:
    """线程安全的环形缓冲区"""
    cap = 3s × 16kHz = 48000 samples
```

**设计目的：**
- 唤醒命中后回灌最近2.5秒音频给ASR
- 解决"首句截断"问题
- 保证用户说话开头不被丢失

**读写机制：**
- 写指针 (`wpos`) 循环递增
- 读时计算起始位置: `(wpos - n) % cap`
- 线程锁保护并发访问

### 3.4 唤醒仲裁器 (WakeArbiter)

#### 3.4.1 动态阈值策略

```python
def dynamic_threshold(snr_db: float) -> float:
    if snr >= 20dB:    t = base_thresh      # 0.35
    elif snr >= 10dB:  t = base_thresh + 0.15
    else:              t = base_thresh + 0.30  # 噪声大时更严格
    
    if tts_playing:    t += 0.10          # TTS期间防止自激
    t += cooldown_bonus                   # 冷却期加成
    return min(t, 0.95)
```

#### 3.4.2 硬否决规则

| 场景 | 规则 | 原因 |
|------|------|------|
| 静默异常 | SNR < -3dB 但分数高 | 异常信号 |
| VAD未激活 | 分数 < 阈值 + 0.10 | 抑制瞬态噪声 |
| 冷却期 | 5秒内触发≥3次 | 防止连续误触发 |

#### 3.4.3 冷却机制

```python
def register_hit(t: float):
    recent = [x for x in hit_times if t - x <= 5.0]
    if len(recent) >= 3:
        cooldown_bonus = 0.15      # 阈值+0.15
        cooldown_until = t + 300.0 # 持续5分钟
```

## 5. 三种触发模式

### 5.1 模式A: KWS (唤醒词模式)

**适用场景:** 传统唤醒词触发（如"Hey Jarvis"）

**处理流程:**

```python
def _process_kws(frame, is_speech, snr, feats):
    # 1. 累积80ms块 (openWakeWord要求)
    kws_accum = concatenate([kws_accum, frame])
    if len(kws_accum) >= 1280:  # 80ms @ 16kHz
        chunk = kws_accum[:1280]
        kws_accum = kws_accum[1280:]
        
        # 2. 调用openWakeWord推理
        preds = kws.predict(chunk)  # {word: score}
        best_name, best_score = max(preds.items(), key=lambda x: x[1])
        
        # 3. 仲裁器判断
        decision = arbiter.decide(best_score, snr, is_speech)
        
        # 4. 触发或丢弃
        if decision.fire and state == IDLE:
            _fire_wake(best_name, best_score)
```

**特点:**
- 使用预训练ONNX模型
- 支持多唤醒词同时监听
- 动态阈值随SNR调整

### 5.2 模式B: VAD (无唤醒词模式)

**适用场景:** 直接说话触发，无需唤醒词

**处理流程:**

```python
def _process_vad(is_speech, snr):
    # 1. 累积语音时长
    if is_speech:
        speech_run_ms += 20
        silence_run_ms = 0
    else:
        silence_run_ms += 20
        if silence_run_ms >= 300ms:
            speech_run_ms = 0  # 长静音清零
    
    # 2. 计算伪分数
    pseudo_score = min(1.0, speech_run_ms / min_speech_ms)
    
    # 3. 仲裁器 + 硬门限
    decision = arbiter.decide(pseudo_score, snr, is_speech)
    
    # 4. 额外硬否决
    if snr < min_snr_db: fire = False  # SNR不足
    if now - last_fire < 1.5s: fire = False  # 防抖
    if speech_run_ms < min_speech_ms: fire = False  # 时长不足
    
    # 5. 触发
    if fire and state == IDLE:
        _fire_wake("(vad-intent)", pseudo_score)
```

**特点:**
- 简单直接，仅需音频
- 误触发风险较高（无唤醒词过滤）
- 适合安静环境或作为对照组

### 5.3 模式C: 多模态模式（默认/推荐）

**适用场景:** 高精度办公场景，防同事语音误触发

**核心设计: 9条硬门限全部命中才触发**

```python
def _process_multimodal(is_speech, snr, feats):
    # ========== 门限1-3: 音频意图证据 ==========
    voiced = feats.is_voiced                    # 浊音检测
    snr_ok = snr >= min_snr_db                  # SNR门限
    speech_run = speech_run_ms >= min_speech_ms # 连续语音时长
    # (带240ms hangover，防"嗯...你好"被截断)
    
    # ========== 门限4-5: 空间定位 ==========
    near_field = feats.near_field_ok or visual.near_field  # 近场
    doa_ok = abs(feats.doa_deg) <= doa_cone_deg  # 方向在锥内
    
    # ========== 门限6-8: 视觉生理证据 ==========
    face = visual.face_present      # 检测到人脸
    gaze = visual.gaze_aligned      # 注视设备
    lip = visual.lip_moving         # 嘴唇在动
    
    # ========== 门限9: 唇-声同步（关键反同事语音） ==========
    # 滑动窗口统计最近800ms
    coincide = sum(1 for v,l in av_window if v and l)  # 同时发声+唇动
    voiced_cnt = sum(1 for v,_ in av_window if v)       # 仅音频发声
    lip_cnt = sum(1 for _,l in av_window if l)          # 仅嘴唇动
    
    av_sync = (
        voiced_cnt >= 4 and lip_cnt >= 4
        and coincide >= max(3, 0.35 * voiced_cnt)
    )
    # 原理: 同事说话时麦克风有声(voiced)但用户嘴没动(lip=0)
    
    # ========== 持续判定 (SUSTAIN) ==========
    all_pass = all([voiced, snr_ok, speech_run, near_field, 
                    doa_ok, face, gaze, lip, av_sync])
    
    if all_pass:
        sustain_pass_frames += 1
    else:
        sustain_pass_frames = 0
    
    sustained = sustain_pass_frames >= 7  # 140ms持续全绿
    
    # ========== 防抖/冷却/TTS抑制 ==========
    if now - last_fire < 1.5s: sustained = False  # 防抖
    if tts_playing and snr < min_snr + 3: sustained = False  # TTS抑制
    
    # ========== 触发 ==========
    if sustained and state == IDLE:
        last_fire = now
        speech_run_ms = 0
        sustain_pass_frames = 0
        _fire_wake("(multimodal)", pseudo_score)
```

**9条硬门限总览:**

| # | 门限 | 来源 | 作用 |
|---|------|------|------|
| 1 | voiced | 音频前端 | 浊音检测，过滤瞬态噪声 |
| 2 | snr | 噪声跟踪器 | 信噪比足够 |
| 3 | speech_run | 时长累积 | 连续说话足够时长 |
| 4 | near_field | 音频/视觉 | 距离足够近 |
| 5 | doa | GCC-PHAT | 方向正确 |
| 6 | face | 视觉 | 检测到人脸 |
| 7 | gaze | 视觉 | 注视设备 |
| 8 | lip | 视觉 | 嘴唇在动 |
| 9 | av_sync | 音视频同步 | 声唇一致（反同事语音） |

## 6. 多模态硬门限机制深度解析

### 6.1 为什么是9条硬门限？

**误触发场景与防线对应:**

| 误触发场景 | 触发门限 | 防线 |
|-----------|----------|------|
| 同事在背后说话 | av_sync | 用户嘴没动 |
| 同事在旁边说话 | near_field + doa | 距离远/方向偏 |
| 用户看别处说话 | gaze | 未注视设备 |
| 键盘/鼠标敲击 | voiced | 非浊音 |
| 空调/风扇噪声 | snr + voiced | 稳态噪声SNR低 |
| 电视BGM | av_sync + speech_run | 无唇动/时长短 |
| TTS自激回环 | tts_playing + snr | TTS期间提高阈值 |

### 6.2 唇-声同步 (AV Sync) 详解

**核心问题:** 同事在旁边说话，麦克风能录到声音，但如何判断不是用户在说话？

**解决方案:**

```
用户场景：                     同事场景：
音频 ◄── 用户说话              音频 ◄── 同事说话
  │         │                    │
  │         ▼                    │
  │    嘴动(lip=True)            │    用户嘴不动(lip=False)
  │         │                    │
  ▼         ▼                    ▼
voiced=True lip=True        voiced=True lip=False
  │                              │
  └──► coincide=1                └──► coincide=0
       av_sync=True                   av_sync=False
       (触发)                         (阻断)
```

**统计窗口设计:**
- 窗口大小: 40帧 × 20ms = 800ms
- 共现要求: `coincide ≥ max(3, 0.35 × voiced_cnt)`
- 反证限制: `voiced_only ≤ coincide`

### 6.3 持续判定 (SUSTAIN) 机制

**问题:** 瞬时全绿可能由噪声尖峰导致

**解决:** 要求连续7帧(140ms)全部通过

```
时间:  0   20  40  60  80  100 120 140 160ms
门限:  N   Y   Y   Y   Y   Y   Y   Y   Y
       │   │   │   │   │   │   │   │
       └───┴───┴───┴───┴───┴───┘   │
           sustain_frames=7         │
                                    ▼
                                FIRE!
```

### 6.4 语音时长累积的Hangover设计

**问题:** 用户说话有停顿，如"嗯...那个..."

**解决:** 浊音后保留12帧(240ms)的半速累积

```
语音:    ████    ████      ████████
         │       │          │
         ▼       ▼          ▼
voiced:  YYYYHHHHYYYYHHHHHHHHYYYY  (H=hangover半速累积)
speech:  ▓▓▓▓░░▓▓▓▓░░░░░░░░▓▓▓▓▓▓▓▓
         │       │          │
         └───────┴──────────┘
              总时长达到阈值 → 触发
```

## 7. ASR语音识别流程

### 7.1 懒加载机制

```python
def _get_asr(self):
    with self._asr_lock:
        if self._asr_model is not None:
            return self._asr_model, self._asr_backend
        
        # 优先级1: SenseVoiceSmall (中文识别更准)
        if 模型文件存在:
            加载 sherpa_onnx.OfflineRecognizer
            return model, "sensevoice"
        
        # 优先级2: faster-whisper (fallback)
        加载 WhisperModel("tiny", device="cpu", compute_type="int8")
        return model, "whisper"
```

### 7.2 转写流程

```python
def _transcribe_async(pcm_i16, name, score, wake_count):
    def _job():
        # 1. 转换为float32并峰值归一化
        audio_f = pcm_i16.astype(np.float32) / 32768.0
        if peak < 0.5:
            audio_f *= min(3.0, 0.8 / peak)  # 提升低音量识别率
        
        # 2. 调用ASR
        if backend == "sensevoice":
            s = model.create_stream()
            s.accept_waveform(SAMPLE_RATE, audio_f)
            model.decode_stream(s)
            text = s.result.text
        else:  # whisper
            segments, info = model.transcribe(
                audio_f, 
                language="zh",
                beam_size=5,
                vad_filter=False,  # 已用VAD门控
                condition_on_previous_text=False,  # 每句独立
                initial_prompt="以下是用户对智能助手的中文指令。"
            )
            text = "".join(seg.text for seg in segments)
        
        # 3. 有效性验证
        rejected, reason = _asr_is_meaningless(text)
        
        # 4. 处理结果
        if rejected:
            _reject_wake(wake_count)  # 回滚唤醒计数
        
        _emit("asr", text=text, rejected=rejected, reason=reason)
    
    threading.Thread(target=_job, daemon=True).start()
```

### 7.3 有效性验证 (_asr_is_meaningless)

**过滤规则:**

| 规则 | 示例 | 判定 |
|------|------|------|
| 空文本 | "" | 无效 |
| 仅标点/空白 | "，。！" | 无效 |
| 全是语气词 | "嗯啊哦呃" | 无效 |
| 长度过短 | "嗯" | 无效 |
| 有意义文本 | "打开灯" | 有效 |

### 10.5 有效性回滚机制

```python
def _reject_wake(self, cnt: int):
    """ASR判定为无效时回退唤醒计数"""
    if self.wake_count == cnt and cnt > 0:
        self.wake_count = cnt - 1
    if self.state == "LISTENING":
        self.state = "IDLE"
        self._emit("state", state=self.state, reason="asr_rejected")
```
